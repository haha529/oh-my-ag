# Сценарий: Мульти-Агентный Проект

## Когда Использовать

Ваша фича охватывает несколько доменов — бэкенд API + фронтенд UI + схема БД + ревью QA. Один агент не справится со всем, и вы хотите, чтобы они работали параллельно.

## Последовательность Координации

```text
/plan → /coordinate → agent:spawn (parallel) → /review → merge
```

1. **`/plan`** — PM-агент декомпозирует фичу на доменные задачи
2. **`/coordinate`** — Определяет порядок выполнения и ответственность
3. **`agent:spawn`** — Агенты выполняют параллельно
4. **`/review`** — QA проверяет кросс-доменную согласованность

## Стратегия Сессии

Используйте один session ID на фичу:

```text
session-auth-v2
```

Назначайте workspace по доменам:

| Агент | Workspace |
|-------|-----------|
| backend | `./apps/api` |
| frontend | `./apps/web` |
| mobile | `./apps/mobile` |

## Пример Запуска

```bash
oma agent:spawn backend "Implement JWT auth API + refresh flow" session-auth-v2 -w ./apps/api &
oma agent:spawn frontend "Build login + refresh UX with error states" session-auth-v2 -w ./apps/web &
oma agent:spawn qa "Review auth risks, test matrix, and regression scope" session-auth-v2 &
wait
```

## Правило «Сначала Контракт»

Прежде чем агенты начнут кодить параллельно, **зафиксируйте контракты API**:

- Схемы запросов/ответов
- Коды и сообщения ошибок
- Допущения жизненного цикла auth/сессии

Если контракты меняются в процессе, приостановите downstream-агентов и переиздайте их промпты с обновлёнными контрактами.

## Контрольные Точки Слияния

Не сливайте, пока:

1. Доменные тесты проходят
2. Точки интеграции соответствуют согласованным контрактам
3. QA-проблемы high/critical решены (или явно отклонены)
4. Changelog обновлён, если изменилось внешне видимое поведение

## Чего НЕ Делать

- Использовать один workspace для всех агентов (кошмар конфликтов слияния)
- Менять контракты, не сообщая другим агентам
- Сливать бэкенд и фронтенд независимо до проверки совместимости

## Когда Готово

- Все запланированные задачи выполнены по всем доменам
- Кросс-доменная интеграция валидирована
- Одобрение QA зафиксировано
