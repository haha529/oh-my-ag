# Мониторинг через Дашборд

Когда у вас несколько агентов работают параллельно, вы хотите видеть, что происходит. Для этого и нужны дашборды.

## Запустить Дашборд

```bash
# UI в терминале
oma dashboard

# Веб UI
oma dashboard:web
# → http://localhost:9847
```

## Рекомендуемая Настройка

Используйте 3 терминала рядом:

| Терминал | Назначение |
|---------|-----------|
| 1 | `oma dashboard` — живой статус агентов |
| 2 | Команды запуска агентов |
| 3 | Логи тестов и сборки |

Держите веб-дашборд открытым в браузере для совместной видимости во время командных сессий.

## Что Вы Видите

Дашборды отслеживают `.serena/memories/` и показывают:

- **Статус сессии** — работает, завершена или упала
- **Доска задач** — какой агент за какую задачу отвечает
- **Прогресс агентов** — количество ходов, текущая активность
- **Результаты** — финальные выходные данные по мере поступления

Обновления событийные (обнаружение изменений файлов) — никаких циклов опроса, съедающих ваш CPU.

## Сигналы для Устранения Неполадок

| Вы Видите | Что Делать |
|----------|-----------|
| «No agents detected» | Проверьте, что агенты запущены с тем же `session-id`. Убедитесь, что `.serena/memories/` записывается. |
| Сессия зависла в «running» | Проверьте метки времени файлов `progress-*`. Перезапустите заблокированных агентов с более чёткими промптами. |
| Частые переподключения (веб) | Проверьте файрвол/прокси. Перезапустите `dashboard:web` и обновите страницу. |
| Отсутствует активность | Убедитесь, что оркестратор пишет в правильную директорию workspace. |

## Перед Слиянием

Быстрый чек-лист по дашборду:

- Все агенты достигли статуса «completed»
- Нет неразрешённых QA-проблем высокой серьёзности
- Файлы результатов присутствуют для каждого агента
- Интеграционные тесты запущены после финальных результатов

## Когда Закончите

Фаза мониторинга завершена, когда:
- Сессия показывает терминальное состояние (completed или stopped)
- История активности объясняет, что произошло
- Вы приняли решение о слиянии/релизе с полной видимостью
