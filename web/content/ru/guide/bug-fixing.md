# Сценарий: Исправление Багов

## Начните с Хорошего Отчёта

Чем лучше ваш баг-репорт, тем быстрее исправление:

```text
Symptom: Login button throws TypeError
Environment: Chrome 130, macOS, production build
Steps to reproduce:
  1. Go to /login
  2. Enter valid credentials
  3. Click "Sign In"
Expected: Redirect to dashboard
Actual: White screen, console shows "Cannot read property 'map' of undefined"
Logs: [paste relevant logs]
```

## Сначала Триаж

| Серьёзность | Что Означает | Ответ |
|------------|-------------|-------|
| **P0** | Потеря данных, обход авторизации, падение продакшена | Бросить всё, привлечь QA/безопасность |
| **P1** | Основной пользовательский поток сломан | Исправить в текущем спринте |
| **P2** | Деградация, но есть обходной путь | Запланировать исправление |
| **P3** | Минорное, не блокирующее | Бэклог |

## Цикл Отладки

1. **Воспроизвести** — точно, в минимальном окружении
2. **Изолировать** — найти корневую причину (не просто симптом)
3. **Исправить** — минимальное безопасное изменение
4. **Протестировать** — регрессионный тест для упавшего пути
5. **Просканировать** — проверить соседний код на такой же паттерн

## Шаблон Промпта

```text
Bug: Login throws "Cannot read property 'map' of undefined"
Repro: Click sign-in with valid credentials
Scope: src/components/auth/*, src/hooks/useAuth.ts
Expected: Redirect to dashboard
Need:
1) root cause analysis
2) minimal fix
3) regression tests
4) scan for similar patterns
```

## Когда Эскалировать

Привлеките QA или безопасность, когда баг затрагивает:

- Аутентификация / сессия / обновление токена
- Границы разрешений
- Платёж / согласованность транзакций
- Производительность под нагрузкой

## После Исправления

Проверьте:
- Оригинальное воспроизведение больше не падает
- Нет новых ошибок в связанных потоках
- Тесты падают до исправления, проходят после
- Путь отката понятен при необходимости
