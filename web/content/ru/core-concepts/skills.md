# Навыки (Skills)

Навыки — это то, что делает каждого агента экспертом. Это структурированные знания — не просто промпты, а протоколы выполнения, шаблоны кода, сценарии обработки ошибок и чек-листы качества.

## Двухуровневый Дизайн

Вот в чём хитрость: навыки не загружают всё сразу. Они используют прогрессивное раскрытие для экономии ~75% токенов.

### Уровень 1: SKILL.md (~800 байт)

Загружается всегда. Содержит:
- Идентичность и роль агента
- Когда активироваться (условия маршрутизации)
- Основные правила и ограничения
- Чего НЕ делать

### Уровень 2: resources/ (загружается по требованию)

Загружается только когда агент реально работает. Содержит глубокий контент:

| Ресурс | Что Делает |
|--------|-----------|
| `execution-protocol.md` | Пошаговый workflow: Анализ → Планирование → Реализация → Проверка |
| `tech-stack.md` | Детальные спецификации технологий и версий |
| `error-playbook.md` | Что делать при ошибках (с эскалацией «3 попытки») |
| `checklist.md` | Доменные проверки качества |
| `snippets.md` | Готовые паттерны кода |
| `examples/` | Примеры входных/выходных данных (few-shot) |

### Как Это Выглядит

```
.agents/skills/oma-frontend/
├── SKILL.md                          ← Загружается всегда (~800 байт)
└── resources/
    ├── execution-protocol.md         ← По требованию
    ├── tech-stack.md
    ├── tailwind-rules.md
    ├── component-template.tsx
    ├── snippets.md
    ├── error-playbook.md
    ├── checklist.md
    └── examples/
```

## Общие Ресурсы

Все агенты разделяют общие основы из `.agents/skills/_shared/`:

| Ресурс | Назначение |
|--------|-----------|
| `skill-routing.md` | Направляет задачи к нужному агенту |
| `context-loading.md` | Какие ресурсы загружать для какого типа задач |
| `prompt-structure.md` | Цель → Контекст → Ограничения → Готово Когда |
| `clarification-protocol.md` | Когда спрашивать, а когда просто предполагать |
| `context-budget.md` | Эффективное чтение файлов по уровням моделей |
| `difficulty-guide.md` | Оценка задачи: Простая / Средняя / Сложная |
| `reasoning-templates.md` | Шаблоны структурированного рассуждения |
| `quality-principles.md` | Универсальные стандарты качества |
| `vendor-detection.md` | Определение текущей IDE/CLI |

## Условные Ресурсы

Некоторые ресурсы загружаются только при определённых условиях:

| Ресурс | Когда Загружается |
|--------|------------------|
| `quality-score.md` | Запрошена оценка качества |
| `experiment-ledger.md` | Применяется экспериментальный подход |
| `exploration-loop.md` | Идёт итеративное исследование |

## Выполнение для Конкретного Вендора

Каждая поддерживаемая CLI имеет свой протокол выполнения в `.agents/skills/_shared/runtime/execution-protocols/`:
- `claude.md` — Паттерны для Claude
- `gemini.md` — Паттерны для Gemini
- `codex.md` — Паттерны для Codex
- `qwen.md` — Паттерны для Qwen

## Почему Это Важно

Без прогрессивного раскрытия загрузка 5 агентов заполнила бы ваше окно контекста ещё до начала работы. С ним вы получаете лёгкую начальную загрузку и глубокое выполнение, когда это действительно нужно.
