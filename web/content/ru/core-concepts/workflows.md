# Рабочие Процессы (Workflows)

Workflows — это структурированные последовательности, которые делают oh-my-agent чем-то большим, чем коллекция промптов. Введите слэш-команду, и запустится многоэтапный процесс с определёнными фазами, контрольными точками качества и критериями завершения.

## Все Команды Workflows

### Крупные Workflows Оркестрации

Они **персистентные** — продолжают работать, пока все задачи не завершены, и не могут быть случайно прерваны.

| Команда | Что Делает |
|---------|-----------|
| `/orchestrate` | Запускает агентов параллельно через CLI, координирует через память, мониторит прогресс, запускает проверку. Режим «сделай всё». |
| `/coordinate` | Пошаговая мультидоменная координация. PM планирует первым, затем агенты выполняют с перекрёстным ревью и циклами QA. |
| `/ultrawork` | Workflow, одержимый качеством. 5 фаз, 11 шагов ревью из 17. Планировать → Реализовать → Проверить → Улучшить → Отправить. |

### Планирование и Исследование

| Команда | Что Делает |
|---------|-----------|
| `/plan` | Декомпозиция задач через PM. Выводит структурированный план в `.agents/plan.json`. |
| `/exec-plan` | Берёт план из `/plan` и выполняет его шаг за шагом. |
| `/brainstorm` | Свободная генерация идей. Исследуйте подходы перед фиксацией реализации. |
| `/deepinit` | Полная инициализация проекта — анализирует кодовую базу, настраивает конвенции, конфигурирует инструменты. |

### Качество и Ревью

| Команда | Что Делает |
|---------|-----------|
| `/review` | QA-ревью: безопасность OWASP, производительность, доступность. Делегирует агенту qa-reviewer. |
| `/debug` | Структурированная отладка: воспроизвести → диагностировать → исправить → регрессионный тест. |

### Дизайн

| Команда | Что Делает |
|---------|-----------|
| `/design` | 7-фазный workflow дизайна. Создаёт DESIGN.md с токенами, паттернами компонентов, правилами доступности и спецификациями для передачи. |

### Утилиты

| Команда | Что Делает |
|---------|-----------|
| `/commit` | Анализирует ваши изменения и создаёт conventional commit с правильным type/scope. |
| `/setup` | Интерактивная настройка проекта. |
| `/tools` | Управление подключениями серверов MCP. |
| `/stack-set` | Установка предпочтений tech stack. |

## Слэш-команды Нужны Не Всегда

oh-my-agent определяет ключевые слова в вашей естественной речи и автоматически активирует workflows. Скажите «запланируй фичу аутентификации» — и workflow планирования запустится без `/plan`.

Это работает на **11 языках** (английский, корейский, японский, китайский, испанский, французский, немецкий, португальский, русский, нидерландский, польский).

Вопросы вроде «что такое orchestrate?» распознаются как информационные и ничего не запускают.

## Skills vs. Workflows

Простое различие:
- **Skills** = экспертиза агента (что агент умеет делать)
- **Workflows** = оркестрированные процессы (как несколько агентов работают вместе)

Навык может быть «создать React-компонент». Workflow — это «запланировать фичу → создать компоненты → проверить безопасность → закоммитить код».

## Типичные Последовательности

### Быстрая Фича
```
/plan → проверить результат → /exec-plan
```

### Сложный Мультидоменный Проект
```
/coordinate → PM планирует → агенты запускаются → QA проверяет → исправить проблемы → отправить
```

### Доставка Максимального Качества
```
/ultrawork → 5-фазный процесс с 11 контрольными точками ревью
```

### Исправление Бага
```
/debug → воспроизвести → корневая причина → исправить → регрессионный тест
```
