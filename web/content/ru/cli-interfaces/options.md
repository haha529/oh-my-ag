# Опции CLI

## Глобальные Опции

Доступны в каждой команде:

| Опция | Что Делает |
|-------|-----------|
| `-h, --help` | Показать справку |
| `-V, --version` | Показать номер версии |

## Опции Вывода

Многие команды поддерживают машиночитаемый вывод:

| Опция | Что Делает |
|-------|-----------|
| `--json` | Вывод в формате JSON |
| `--output <format>` | Формат вывода: `text` или `json` |

Также можно установить переменную окружения `OH_MY_AG_OUTPUT_FORMAT=json`.

**Поддерживается:** `doctor`, `stats`, `retro`, `cleanup`, `auth:status`, `usage:anti`, `memory:init`, `verify`, `visualize`

## Опции По Командам

### `update`
| Опция | Что Делает |
|-------|-----------|
| `-f, --force` | Перезаписать пользовательские конфигурации |
| `--ci` | Неинтерактивный режим (пропустить все промпты) |

### `stats`
| Опция | Что Делает |
|-------|-----------|
| `--reset` | Сбросить все данные метрик |

### `retro`
| Опция | Что Делает |
|-------|-----------|
| `--interactive` | Режим ручного ввода |
| `--compare` | Сравнить текущее окно с предыдущим такой же длины |

### `cleanup`
| Опция | Что Делает |
|-------|-----------|
| `--dry-run` | Показать, что будет очищено, без выполнения |
| `-y, --yes` | Пропустить промпты подтверждения |

### `usage:anti`
| Опция | Что Делает |
|-------|-----------|
| `--raw` | Вывести сырой ответ RPC |

### `agent:spawn`
| Опция | Что Делает |
|-------|-----------|
| `-v, --vendor <vendor>` | Переопределить CLI-вендор (`gemini`/`claude`/`codex`/`qwen`) |
| `-w, --workspace <path>` | Рабочая директория для агента |

### `agent:status`
| Опция | Что Делает |
|-------|-----------|
| `-r, --root <path>` | Корневой путь для проверки памяти |

### `agent:parallel`
| Опция | Что Делает |
|-------|-----------|
| `-v, --vendor <vendor>` | Переопределить CLI-вендор |
| `-i, --inline` | Указать задачи как аргументы `agent:task` |
| `--no-wait` | Не ждать завершения (фоновый режим) |

### `memory:init`
| Опция | Что Делает |
|-------|-----------|
| `--force` | Перезаписать существующие файлы схемы |

### `verify`
| Опция | Что Делает |
|-------|-----------|
| `-w, --workspace <path>` | Путь workspace для проверки |

## Практические Примеры

```bash
# JSON-вывод для CI-пайплайна
oma doctor --json

# Сбросить метрики продуктивности
oma stats --reset

# Предпросмотр очистки без выполнения
oma cleanup --dry-run

# Запуск с конкретной CLI и workspace
oma agent:spawn backend "Auth API" session-01 -v codex -w ./apps/api

# Неинтерактивное обновление в CI
oma update --ci --force

# Сравнить последние 7 дней с предыдущими 7 днями
oma retro 7 --compare
```
