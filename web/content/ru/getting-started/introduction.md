# Введение

Когда-нибудь хотели, чтобы у вашего ИИ-помощника были коллеги? Именно это делает oh-my-agent.

Вместо одного ИИ, который делает всё (и путается на полпути), oh-my-agent распределяет работу между **специализированными агентами** — эксперт по фронтенду, эксперт по бэкенду, ревьюер QA, планировщик PM и другие. Каждый агент глубоко знает свою область и имеет собственные инструменты, чек-листы и сценарии обработки ошибок.

## Что Вы Получаете

- **14 специализированных агентов** — каждый смоделирован по реальной роли в инженерной команде
- **Слэш-команды** вроде `/plan`, `/coordinate`, `/review`, которые запускают структурированные рабочие процессы
- **Автоматическое определение** — просто опишите, что хотите, на естественном языке, и нужный workflow активируется (на 11 языках!)
- **Параллельное выполнение** — запускайте несколько агентов, работающих над разными частями одновременно
- **Дашборды в реальном времени** — терминал и веб-интерфейс для мониторинга работы агентов
- **Работает везде** — Antigravity, Claude Code, Cursor, Gemini CLI, Codex CLI, OpenCode и другие

## Познакомьтесь с Командой

| Агент | Что Делает |
|-------|-----------|
| **oma-brainstorm** | Исследует идеи до того, как вы начнёте что-то строить |
| **oma-pm** | Декомпозирует требования, планирует задачи, определяет контракты API |
| **oma-frontend** | Создаёт UI на React/Next.js с TypeScript и Tailwind CSS |
| **oma-backend** | Создаёт API на Python, Node.js или Rust |
| **oma-db** | Проектирует схемы БД, управляет миграциями, оптимизирует запросы |
| **oma-mobile** | Создаёт кроссплатформенные приложения на Flutter |
| **oma-design** | Создаёт дизайн-системы с токенами, доступностью и адаптивными правилами |
| **oma-qa** | Проверяет безопасность (OWASP), производительность и доступность |
| **oma-debug** | Находит корневые причины, пишет исправления и регрессионные тесты |
| **oma-tf-infra** | Разворачивает облачную инфраструктуру с Terraform |
| **oma-dev-workflow** | Автоматизирует CI/CD, релизы и задачи монорепозитория |
| **oma-translator** | Естественно переводит контент между языками |
| **oma-orchestrator** | Запускает несколько агентов параллельно через CLI |
| **oma-commit** | Создаёт чистые conventional commits |

## Как Это Работает (30-Секундная Версия)

1. Вы описываете, что хотите построить
2. oh-my-agent определяет, какие агенты нужны
3. Агенты выполняют работу со своими специализированными знаниями
4. Контрольные точки качества проверяют работу перед завершением
5. Вы получаете код, готовый к продакшену, а не просто предложения

## Ключевая Идея

Всё живёт в `.agents/` — единственная портативная директория в вашем проекте. Навыки, рабочие процессы, конфигурации — всё здесь. Меняйте IDE когда угодно. Ваша настройка агентов путешествует вместе с кодом.
