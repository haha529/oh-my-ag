# はじめに

AIアシスタントに「同僚」がいたらいいのに、と思ったことはありませんか？ それを実現するのがoh-my-agentです。

1つのAIにすべてを任せる（そして途中で混乱する）代わりに、oh-my-agentは作業を**専門エージェント**に分割します。フロントエンド専門家、バックエンド専門家、QAレビュアー、PMプランナーなど、それぞれのエージェントが自分の領域を深く理解しており、独自のツール、チェックリスト、エラー対応プレイブックを持っています。

## 何が手に入るのか

- **14の専門エージェント** — 実際のエンジニアリングチームの役割をモデル化
- **スラッシュコマンド** — `/plan`、`/coordinate`、`/review` など、構造化されたワークフローを起動
- **自動検出** — 自然言語で説明するだけで、適切なワークフローが自動起動（11言語対応！）
- **並列実行** — 複数のエージェントを同時に別々の作業に投入
- **リアルタイムダッシュボード** — ターミナルとWeb UIでエージェントの動作を監視
- **どこでも動作** — Antigravity、Claude Code、Cursor、Gemini CLI、Codex CLI、OpenCodeなど対応

## チームメンバー紹介

| エージェント | 役割 |
|-------|-------------|
| **oma-brainstorm** | 実装を始める前にアイデアを探索 |
| **oma-pm** | 要件分解、タスク計画、APIコントラクト定義 |
| **oma-frontend** | React/Next.jsのUIをTypeScriptとTailwind CSSで構築 |
| **oma-backend** | Python、Node.js、RustでAPIを作成 |
| **oma-db** | データベーススキーマ設計、マイグレーション、クエリチューニング |
| **oma-mobile** | Flutterでクロスプラットフォームアプリを構築 |
| **oma-design** | デザイントークン、アクセシビリティ、レスポンシブルールを含むデザインシステムを作成 |
| **oma-qa** | セキュリティ（OWASP）、パフォーマンス、アクセシビリティをレビュー |
| **oma-debug** | 根本原因を特定し、修正とリグレッションテストを作成 |
| **oma-tf-infra** | Terraformでクラウドインフラをプロビジョニング |
| **oma-dev-workflow** | CI/CD、リリース、モノレポタスクを自動化 |
| **oma-translator** | コンテンツを自然に多言語翻訳 |
| **oma-orchestrator** | CLIから複数エージェントを並列実行 |
| **oma-commit** | クリーンなConventional Commitsを作成 |

## 仕組み（30秒バージョン）

1. 作りたいものを説明する
2. oh-my-agentが必要なエージェントを判断する
3. エージェントが専門知識を使って実行する
4. 品質ゲートが完了前に作業をチェックする
5. 提案ではなく、本番対応のコードが得られる

## 重要なコンセプト

すべてが `.agents/` に収まっています。プロジェクト内のポータブルなディレクトリ1つに、スキル、ワークフロー、設定がすべて入っています。IDEをいつ切り替えても大丈夫。エージェントの設定はコードと一緒に移動します。
