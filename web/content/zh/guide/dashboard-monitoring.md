# 仪表盘监控

当你有多个智能体并行运行时，你需要随时了解发生了什么。这就是仪表盘的作用。

## 启动仪表盘

```bash
# 终端界面
oma dashboard

# Web 界面
oma dashboard:web
# → http://localhost:9847
```

## 推荐设置

并排使用 3 个终端：

| 终端 | 用途 |
|------|------|
| 1 | `oma dashboard` —— 实时智能体状态 |
| 2 | 智能体启动命令 |
| 3 | 测试和构建日志 |

团队协作时，在浏览器中打开 Web 仪表盘以便共享查看。

## 你会看到什么

仪表盘监视 `.serena/memories/` 并显示：

- **会话状态** —— 运行中、已完成或失败
- **任务看板** —— 哪个智能体负责哪个任务
- **智能体进度** —— 执行轮次、当前活动
- **结果** —— 最终输出实时到达

更新是事件驱动的（文件变更检测）—— 不会有轮询循环消耗你的 CPU。

## 故障排除信号

| 看到的情况 | 怎么办 |
|-----------|--------|
| "No agents detected" | 检查智能体是否使用了相同的 `session-id` 启动。确认 `.serena/memories/` 正在被写入。 |
| 会话卡在"running"状态 | 检查 `progress-*` 文件的时间戳。用更清晰的提示重新启动被阻塞的智能体。 |
| 频繁重连（Web） | 检查防火墙/代理。重启 `dashboard:web` 并刷新页面。 |
| 缺少活动记录 | 确认编排器正在向正确的工作区目录写入。 |

## 合并前检查

通过仪表盘快速确认：

- 所有智能体达到"completed"状态
- 没有未解决的高严重级别 QA 发现
- 每个智能体都有对应的结果文件
- 最终输出后运行了集成测试

## 完成标志

监控阶段在以下情况完成：
- 会话显示终态（completed 或 stopped）
- 活动历史记录了发生的事情
- 你在充分了解情况后做出了合并/发布决策
