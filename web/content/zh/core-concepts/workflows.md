# 工作流

工作流是让 oh-my-agent 超越普通提示词集合的结构化流程。输入一个斜杠命令，就会启动一个包含明确阶段、质量关卡和完成标准的多步骤过程。

## 所有工作流命令

### 大型编排工作流

这些是**持久性**的 —— 它们会持续运行直到所有任务完成，不会被意外中断。

| 命令 | 功能 |
|------|------|
| `/orchestrate` | 通过 CLI 并行启动智能体，通过内存协调，监控进度，运行验证。"全自动"模式。 |
| `/coordinate` | 逐步的多领域协调。PM 先规划，然后智能体执行，包含交叉审查和 QA 循环。 |
| `/ultrawork` | 追求极致质量的工作流。5 个阶段，17 步中有 11 步是审查。规划 → 实现 → 验证 → 优化 → 交付。 |

### 规划与探索

| 命令 | 功能 |
|------|------|
| `/plan` | PM 驱动的任务分解。输出结构化计划到 `.agents/plan.json`。 |
| `/exec-plan` | 接收 `/plan` 的计划并逐步执行。 |
| `/brainstorm` | 自由形式的头脑风暴。在动手实现前探索各种方案。 |
| `/deepinit` | 完整的项目初始化 —— 分析代码库、建立规范、配置工具。 |

### 质量与审查

| 命令 | 功能 |
|------|------|
| `/review` | QA 审查：OWASP 安全性、性能、无障碍。委托给 qa-reviewer 智能体。 |
| `/debug` | 结构化调试：复现 → 诊断 → 修复 → 回归测试。 |

### 设计

| 命令 | 功能 |
|------|------|
| `/design` | 7 阶段设计工作流。创建包含令牌、组件模式、无障碍规则和交付规格的 DESIGN.md。 |

### 实用工具

| 命令 | 功能 |
|------|------|
| `/commit` | 分析你的变更并创建带有正确 type/scope 的 conventional commit。 |
| `/setup` | 交互式项目配置。 |
| `/tools` | 管理 MCP 服务器连接。 |
| `/stack-set` | 设置技术栈偏好。 |

## 不一定需要斜杠命令

oh-my-agent 能检测你自然语言中的关键词并自动激活工作流。说"规划认证功能"，计划工作流就会启动 —— 不需要输入 `/plan`。

支持 **11 种语言**（英语、韩语、日语、中文、西班牙语、法语、德语、葡萄牙语、俄语、荷兰语、波兰语）。

像"什么是 orchestrate？"这样的提问会被识别为信息查询，不会触发任何工作流。

## 技能 vs. 工作流

简单区分：
- **技能** = 智能体的专长（智能体知道怎么做什么）
- **工作流** = 编排好的流程（多个智能体如何协同工作）

技能可能是"构建一个 React 组件"。工作流则是"规划功能 → 构建组件 → 审查安全性 → 提交代码"。

## 典型使用序列

### 快速功能开发
```
/plan → 审查输出 → /exec-plan
```

### 复杂的多领域项目
```
/coordinate → PM 规划 → 智能体启动 → QA 审查 → 修复问题 → 交付
```

### 最高质量交付
```
/ultrawork → 5 阶段关卡流程，含 11 个审查检查点
```

### Bug 修复
```
/debug → 复现 → 根因分析 → 修复 → 回归测试
```
