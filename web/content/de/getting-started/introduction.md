# Einfuehrung

Hast du dir schon mal gewuenscht, dein KI-Assistent haette Kollegen? Genau das macht oh-my-agent.

Statt dass eine einzige KI alles erledigt (und dabei auf halbem Weg durcheinander kommt), verteilt oh-my-agent die Arbeit auf **spezialisierte Agenten** — einen Frontend-Experten, einen Backend-Experten, einen QA-Reviewer, einen PM-Planer und mehr. Jeder Agent kennt seinen Bereich genau und hat eigene Werkzeuge, Checklisten und Fehler-Playbooks.

## Was Du Wirklich Bekommst

- **14 spezialisierte Agenten** — jeder nach einer echten Engineering-Team-Rolle modelliert
- **Slash-Befehle** wie `/plan`, `/coordinate`, `/review`, die strukturierte Workflows ausloesen
- **Automatische Erkennung** — beschreib einfach, was du willst, und der richtige Workflow aktiviert sich (in 11 Sprachen!)
- **Parallele Ausfuehrung** — starte mehrere Agenten, die gleichzeitig an verschiedenen Teilen arbeiten
- **Echtzeit-Dashboards** — Terminal und Web-UI, um zu ueberwachen, was deine Agenten tun
- **Funktioniert ueberall** — Antigravity, Claude Code, Cursor, Gemini CLI, Codex CLI, OpenCode und mehr

## Lerne Dein Team Kennen

| Agent | Was Er Tut |
|-------|-----------|
| **oma-brainstorm** | Erkundet Ideen, bevor du dich zum Bauen verpflichtest |
| **oma-pm** | Zerlegt Anforderungen, plant Aufgaben, definiert API-Vertraege |
| **oma-frontend** | Baut React/Next.js UI mit TypeScript und Tailwind CSS |
| **oma-backend** | Erstellt APIs in Python, Node.js oder Rust |
| **oma-db** | Entwirft Datenbank-Schemata, verwaltet Migrationen, optimiert Abfragen |
| **oma-mobile** | Baut plattformuebergreifende Apps mit Flutter |
| **oma-design** | Erstellt Design-Systeme mit Tokens, Barrierefreiheit und responsiven Regeln |
| **oma-qa** | Prueft Sicherheit (OWASP), Performance und Barrierefreiheit |
| **oma-debug** | Findet Grundursachen, schreibt Fixes und Regressionstests |
| **oma-tf-infra** | Provisioniert Cloud-Infrastruktur mit Terraform |
| **oma-dev-workflow** | Automatisiert CI/CD, Releases und Monorepo-Aufgaben |
| **oma-translator** | Uebersetzt Inhalte natuerlich zwischen Sprachen |
| **oma-orchestrator** | Fuehrt mehrere Agenten parallel via CLI aus |
| **oma-commit** | Erstellt saubere konventionelle Commits |

## Wie Es Funktioniert (Die 30-Sekunden-Version)

1. Du beschreibst, was du bauen willst
2. oh-my-agent bestimmt, welche Agenten gebraucht werden
3. Agenten fuehren mit ihrem Spezialwissen aus
4. Qualitaets-Gates pruefen die Arbeit, bevor sie fertig ist
5. Du bekommst produktionsfertigen Code, nicht nur Vorschlaege

## Die Kernidee

Alles lebt in `.agents/` — ein einziges portables Verzeichnis in deinem Projekt. Skills, Workflows, Configs, alles. Wechsle jederzeit die IDE. Dein Agenten-Setup reist mit deinem Code.
